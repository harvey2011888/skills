## 🚀 使用方式

### 方式 1: SSH Key（推荐）

```bash
# 生成 SSH Key（如果还没有）
ssh-keygen -t ed25519 -C "your_email@example.com"

# 将公钥添加到 Git 平台（GitHub/GitLab/Gitee）
# 然后使用：
python cli.py --private-repo --ssh-key ~/.ssh/id_ed25519
```

### 方式 2: Token（推荐）

```bash
# 1. 生成 Personal Access Token
# GitHub: https://github.com/settings/tokens
# GitLab: https://gitlab.com/-/profile/personal_access_tokens
# Gitee: https://gitee.com/profile/personal_access_tokens

# 2. 使用 Token
python cli.py --private-repo \
  --token "ghp_xxxxxxxxxxxxxxxxxxxx" \
  --remote-url "https://github.com/username/private-repo.git"
```

### 方式 3: 配置文件

```bash
# 创建配置文件
cat > config_private.json << 'EOF'
{
  "private_repo": true,
  "git_auth": {
    "auth_type": "token",
    "token": "your_token_here",
    "remote_url": "https://github.com/username/repo.git"
  }
}
EOF

# 使用配置文件
python cli.py --config config_private.json --output weekly_report.md
```

### 方式 4: Python 代码

```python
from weekly_report_generator import ReportGenerator, ReportConfig, GitAuthConfig

# 配置认证
git_auth = GitAuthConfig(
    auth_type="token",
    token="ghp_xxxxxxxxxxxxxxxxxxxx",
    remote_url="https://github.com/username/private-repo.git"
)

# 创建配置
config = ReportConfig(
    report_type="weekly",
    private_repo=True,
    git_auth=git_auth
)

# 生成报告
generator = ReportGenerator(config)
report = generator.generate()
```

## 🔒 安全特性

### 错误处理增强

**认证失败提示**:
```
❌ Git 认证失败：请检查私有仓库认证配置
   错误信息：Authentication failed
   解决方案：
     1. 使用 SSH 方式：配置 SSH Key
     2. 使用 Token 方式：生成 Personal Access Token
     3. 使用 HTTPS 方式：确保用户名密码正确
```

**仓库访问失败提示**:
```
❌ 无法访问私有仓库：/path/to/repo
   请确认仓库路径正确且有访问权限
```

### 安全最佳实践

1. ✅ **不要提交敏感信息**
   ```bash
   # 添加到 .gitignore
   config_private.json
   *.secret.json
   ```

2. ✅ **设置文件权限**
   ```bash
   chmod 600 config_private.json
   chmod 600 ~/.ssh/id_rsa
   ```

3. ✅ **使用环境变量**
   ```bash
   export GIT_TOKEN="your_token"
   python cli.py --private-repo --token $GIT_TOKEN
   ```

4. ✅ **定期更换凭证**
   - Token 建议 30-90 天更换一次
   - 设置 Token 过期时间
