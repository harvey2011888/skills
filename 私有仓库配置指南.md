# 🔐 私有仓库配置指南

## 为什么需要配置认证？

当你的代码存储在**私有 Git 仓库**（如 GitHub Private Repo、GitLab Private Project）时，需要配置认证信息才能访问 Git 提交记录。

## 支持的认证方式

本 Skill 支持三种认证方式：

### 方式一：SSH Key 认证（推荐 ⭐）

**优点**：
- 安全性高
- 一次配置，长期使用
- 无需每次输入密码

**配置步骤**：

1. **生成 SSH Key**（如果已有可跳过）
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```
   或使用 RSA：
   ```bash
   ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
   ```

2. **将公钥添加到 Git 平台**
   - GitHub: Settings → SSH and GPG keys → New SSH key
   - GitLab: Settings → SSH Keys
   - Gitee: 设置 → SSH 公钥

3. **测试连接**
   ```bash
   ssh -T git@github.com
   ```

4. **在 Skill 中使用**
   ```bash
   python cli.py --private-repo --auth-type ssh --ssh-key ~/.ssh/id_ed25519
   ```

**配置文件方式**：
```json
{
  "private_repo": true,
  "git_auth": {
    "auth_type": "ssh",
    "ssh_key_path": "/Users/yourname/.ssh/id_ed25519",
    "remote_url": "git@github.com:username/private-repo.git"
  }
}
```

---

### 方式二：Personal Access Token 认证（推荐 ⭐）

**优点**：
- 易于管理
- 可以设置权限和过期时间
- 适合 CI/CD 场景

**配置步骤**：

#### GitHub
1. 访问 [Personal Access Tokens](https://github.com/settings/tokens)
2. 点击 "Generate new token (classic)"
3. 选择权限：至少勾选 `repo` (Full control of private repositories)
4. 生成并复制 Token

#### GitLab
1. 访问 [Access Tokens](https://gitlab.com/-/profile/personal_access_tokens)
2. 选择权限：`api` 或 `read_repository`
3. 创建并复制 Token

#### Gitee
1. 访问 [个人访问令牌](https://gitee.com/profile/personal_access_tokens)
2. 选择权限：`projects`
3. 创建并复制 Token

**在 Skill 中使用**：
```bash
python cli.py --private-repo \
  --auth-type token \
  --token "ghp_xxxxxxxxxxxxxxxxxxxx" \
  --remote-url "https://github.com/username/private-repo.git"
```

**配置文件方式**：
```json
{
  "private_repo": true,
  "git_auth": {
    "auth_type": "token",
    "token": "ghp_xxxxxxxxxxxxxxxxxxxx",
    "remote_url": "https://github.com/username/private-repo.git"
  }
}
```

**⚠️ 安全提示**：
- Token 具有访问你私有仓库的权限，请妥善保管
- 不要将包含 Token 的配置文件提交到 Git
- 建议设置 Token 过期时间
- 定期更换 Token

---

### 方式三：HTTPS 用户名密码认证

**优点**：
- 配置简单
- 适合临时使用

**缺点**：
- 安全性较低
- 密码可能明文存储

**在 Skill 中使用**：
```bash
python cli.py --private-repo \
  --auth-type https \
  --username "your_username" \
  --password "your_password" \
  --remote-url "https://github.com/username/private-repo.git"
```

**配置文件方式**：
```json
{
  "private_repo": true,
  "git_auth": {
    "auth_type": "https",
    "username": "your_username",
    "password": "your_password",
    "remote_url": "https://github.com/username/private-repo.git"
  }
}
```

**⚠️ 安全提示**：
- 不建议在配置文件中明文存储密码
- 建议使用 Token 方式替代
- 确保配置文件权限设置为仅自己可读

## 使用示例

### 示例 1：使用 SSH Key

```bash
# 使用默认 SSH Key
python cli.py --private-repo --ssh-key ~/.ssh/id_rsa --output weekly_report.md

# 使用自定义 SSH Key
python cli.py --private-repo --ssh-key /path/to/custom/key --output weekly_report.md
```

### 示例 2：使用 Token

```bash
# 命令行指定 Token
python cli.py --private-repo \
  --token "ghp_xxxxxxxxxxxxxxxxxxxx" \
  --remote-url "https://github.com/company/private-project.git" \
  --author "张三" \
  --team "开发部" \
  --output weekly_report.md
```

### 示例 3：使用配置文件

1. 创建配置文件 `config_private.json`：
```json
{
  "private_repo": true,
  "git_auth": {
    "auth_type": "token",
    "token": "your_token_here",
    "remote_url": "https://github.com/username/private-repo.git"
  },
  "author_name": "张三",
  "team_name": "开发团队",
  "highlights": ["完成核心功能"],
  "plans": ["推进新项目"]
}
```

2. 使用配置文件：
```bash
python cli.py --config config_private.json --output weekly_report.md
```

### 示例 4：访问多个私有仓库

为不同的仓库创建不同的配置文件：

```bash
# 公司项目配置
cat > config_work.json << 'EOF'
{
  "private_repo": true,
  "git_auth": {
    "auth_type": "token",
    "token": "work_token_here",
    "remote_url": "https://github.com/company/project.git"
  }
}
EOF

# 个人项目配置
cat > config_personal.json << 'EOF'
{
  "private_repo": true,
  "git_auth": {
    "auth_type": "ssh",
    "ssh_key_path": "~/.ssh/id_rsa",
    "remote_url": "git@github.com:username/personal-project.git"
  }
}
EOF

# 使用不同配置生成报告
python cli.py --config config_work.json --output work_report.md
python cli.py --config config_personal.json --output personal_report.md
```

## 常见问题

### Q1: 提示 "Permission denied (publickey)"

**原因**：SSH Key 未配置或权限不正确

**解决方案**：
1. 检查 SSH Key 是否存在：`ls -la ~/.ssh/`
2. 检查公钥是否添加到 Git 平台
3. 测试 SSH 连接：`ssh -T git@github.com`
4. 确保私钥权限正确：`chmod 600 ~/.ssh/id_rsa`

### Q2: 提示 "Authentication failed"

**原因**：Token 或密码错误

**解决方案**：
1. 检查 Token 是否正确复制（无多余空格）
2. 检查 Token 是否过期
3. 检查 Token 权限是否包含 `repo` 或 `read_repository`
4. 重新生成 Token 并更新配置

### Q3: 提示 "repository not found"

**原因**：仓库 URL 错误或无访问权限

**解决方案**：
1. 检查 remote_url 是否正确
2. 确认 Token/SSH Key 有访问该仓库的权限
3. 如果是组织仓库，确认组织管理员已授权

### Q4: 如何安全地存储 Token？

**推荐方案**：

1. **使用环境变量**
   ```bash
   export GIT_TOKEN="ghp_xxxxxxxxxxxxxxxxxxxx"
   python cli.py --private-repo --token $GIT_TOKEN
   ```

2. **使用配置文件但设置权限**
   ```bash
   chmod 600 config_private.json
   ```

3. **使用密钥管理服务**
   - macOS: Keychain
   - Linux: gnome-keyring
   - Windows: Credential Manager

### Q5: CI/CD 中如何使用？

**GitHub Actions 示例**：
```yaml
name: Weekly Report
on:
  schedule:
    - cron: '0 9 * * 5'

jobs:
  generate-report:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Generate Weekly Report
        env:
          GIT_TOKEN: ${{ secrets.GIT_TOKEN }}
        run: |
          python cli.py --private-repo \
            --token "$GIT_TOKEN" \
            --remote-url "https://github.com/company/private-repo.git" \
            --output weekly_report.md
      
      - name: Upload Report
        uses: actions/upload-artifact@v2
        with:
          name: weekly-report
          path: weekly_report.md
```

## 安全最佳实践

1. ✅ **优先使用 SSH Key** - 比 Token 和密码更安全
2. ✅ **设置 Token 过期时间** - 建议 30-90 天
3. ✅ **最小权限原则** - 只授予必要的权限
4. ✅ **不要提交敏感信息** - 将配置文件添加到 `.gitignore`
5. ✅ **定期更换凭证** - 特别是 Token 和密码
6. ✅ **使用环境变量** - 避免在代码中硬编码凭证

### .gitignore 配置

```gitignore

```

## 故障排查

### 调试模式

添加 `--verbose` 参数查看详细日志：
```bash
python cli.py --private-repo --token "xxx" --verbose
```

### 检查 Git 配置

```bash
# 检查远程仓库 URL
git remote -v

# 测试 Git 连接
git ls-remote origin

# 检查 Git 认证配置
git config --global --list | grep credential
```

### 清除缓存的凭证

```bash
# macOS
git credential-osxkeychain erase

# Windows
git credential-manager erase

# Linux
git credential-cache exit
```
