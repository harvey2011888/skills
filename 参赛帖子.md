# 【Skill 创作】周报/月报自动生成器 - 从 Git、任务、文档中自动提取工作成果

## 1、Skill 简介

这是一个帮助开发者和技术团队**自动生成周报/月报**的 Skill。它能够从 Git 提交记录、任务系统、文档变更中提取工作成果，自动生成结构化的周报或月报，省去了手动整理工作内容的时间。

**适合人群**：需要定期提交周报/月报的开发者、技术团队负责人、项目经理

## 2、使用场景

### 为什么想做它？

作为一名开发者，我深知每周写周报的痛苦：

- 一周下来做了很多事情，但到写周报时却记不全
- 需要手动翻阅 Git 提交记录、任务系统、文档变更记录
- 每周都要花 30-60 分钟整理工作内容，效率很低
- 周报内容容易流于形式，缺乏具体的数据支撑

### 使用前后的对比

**使用前**：
- 每周五下午花 1 小时回忆这周做了什么
- 打开 Git 界面一条条看提交记录
- 去任务系统标记完成的任务
- 整理文档更新记录
- 手动编写周报，格式还不统一

**使用后**：
- 一条命令自动生成周报
- 包含完整的 Git 提交统计
- 任务完成情况自动汇总
- 文档变更自动追踪
- 格式统一，内容完整
- **时间从 60 分钟缩短到 5 分钟**（只需补充亮点和计划）

## 3、创作过程

### 技术选型

考虑到实用性和易用性，我选择了以下技术栈：

- **Python 3.7+** - 跨平台，易于理解和修改
- **Git 命令行** - 直接调用 Git 命令获取提交记录
- **JSON/Markdown** - 支持多种任务文件格式

### 核心架构设计

整个 Skill 分为四个核心模块：

```
周报/月报自动生成器
├── GitExtractor (Git 提交记录提取器)
│   ├── get_commits() - 获取指定日期范围的提交
│   ├── _parse_git_log() - 解析 Git 日志
│   └── get_commit_stats() - 统计提交数据
├── TaskExtractor (任务提取器)
│   ├── extract_from_file() - 从文件提取任务
│   ├── _parse_json_tasks() - 解析 JSON 格式
│   └── _parse_markdown_tasks() - 解析 Markdown 格式
├── DocumentChangeTracker (文档变更追踪器)
│   ├── track_changes() - 追踪文档变更
│   └── categorize_documents() - 分类文档
└── ReportGenerator (报告生成引擎)
    ├── generate() - 生成完整报告
    ├── _generate_git_section() - 生成 Git 部分
    ├── _generate_task_section() - 生成任务部分
    ├── _generate_document_section() - 生作文档部分
    └── _assemble_report() - 组装报告
```

### 关键实现 - Git 提交记录提取

```python
class GitExtractor:
    def get_commits(self, start_date: str, end_date: str) -> List[GitCommit]:
        cmd = [
            "git", "-C", self.repo_path, "log",
            f"--since={start_date}",
            f"--until={end_date}",
            "--pretty=format:%H|%an|%ad|%s",
            "--date=iso",
            "--name-status"
        ]
        result = subprocess.run(cmd, capture_output=True, text=True, check=True)
        return self._parse_git_log(result.stdout)
```

**设计思路**：
1. 使用 `git log` 命令获取指定日期范围的提交
2. 通过 `--pretty=format` 自定义输出格式，便于解析
3. 使用 `--name-status` 获取修改的文件列表
4. 解析输出，构建结构化的提交记录对象

### 关键实现 - 报告生成引擎

```python
class ReportGenerator:
    def generate(self) -> str:
        start_date = self.config.start_date or self._get_default_start_date()
        end_date = self.config.end_date or datetime.now().strftime("%Y-%m-%d")
        
        sections = []
        
        # 收集 Git 提交、任务、文档变更信息
        if self.config.include_git:
            commits = self.git_extractor.get_commits(start_date, end_date)
            stats = self.git_extractor.get_commit_stats(commits)
            sections.append(self._generate_git_section(commits, stats))
        
        # 组装各个部分
        sections.append(self._generate_highlights_section())
        sections.append(self._generate_challenges_section())
        sections.append(self._generate_plans_section())
        
        return self._assemble_report(sections, start_date, end_date)
```

### 配置系统设计

支持多种配置方式：

1. **命令行参数** - 快速指定常用选项
2. **JSON 配置文件** - 保存个性化设置
3. **代码调用** - 灵活集成到其他系统

```json
{
  "report_type": "weekly",
  "author_name": "张三",
  "team_name": "开发团队",
  "highlights": ["完成了核心功能开发"],
  "challenges": ["需要优化数据库查询"],
  "plans": ["推进新功能开发"]
}
```

### 遇到的挑战

1. **Git 日志解析** - 不同格式的提交信息需要健壮的解析逻辑
   - 解决方案：使用分隔符 `|` 分割字段，处理边界情况

2. **任务格式多样化** - 不同团队使用不同的任务管理方式
   - 解决方案：支持 JSON 和 Markdown 两种常见格式，预留扩展接口

3. **日期范围计算** - 周报和月报的日期范围计算规则不同
   - 解决方案：提供自动计算逻辑，也支持手动指定

## 4、使用步骤

### 方式一：命令行工具

```bash
# 1. 生成默认周报（自动计算本周日期范围）
python cli.py

# 2. 生成周报并保存到文件
python cli.py --output weekly_report.md

# 3. 生成月报
python cli.py --type monthly

# 4. 指定日期范围
python cli.py --start-date 2026-05-01 --end-date 2026-05-31

# 5. 使用配置文件
python cli.py --config config.json --output report.md

# 6. 指定作者和团队
python cli.py --author "张三" --team "后端开发组"
```

### 方式二：Python 代码调用

```python
from weekly_report_generator import ReportGenerator, ReportConfig

# 创建配置
config = ReportConfig(
    report_type="weekly",
    author_name="张三",
    team_name="开发团队",
    highlights=[
        "完成了用户认证模块",
        "优化了数据库查询性能"
    ],
    challenges=["跨团队协作需要加强"],
    plans=["推进新功能开发"]
)

# 生成报告
generator = ReportGenerator(config)
report = generator.generate()

# 保存报告
with open("weekly_report.md", "w", encoding="utf-8") as f:
    f.write(report)
```

### 方式三：准备任务文件（可选）

创建 `tasks.json` 文件：

```json
{
  "tasks": [
    {
      "id": "TASK-001",
      "title": "实现用户登录功能",
      "status": "completed",
      "priority": "high",
      "completed_date": "2026-05-10"
    },
    {
      "id": "TASK-002",
      "title": "编写 API 文档",
      "status": "pending",
      "priority": "medium"
    }
  ]
}
```

然后运行：

```bash
python cli.py --config config.json --output weekly_report.md
```
