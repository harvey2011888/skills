## 在 SOLO 中安装步骤

### 步骤 1: 打开 SOLO 技能管理
1. 打开 TRAE SOLO
2. 点击左侧菜单的 **"技能"**
3. 点击右上角的 **"上传技能"** 按钮

### 步骤 2: 上传 Skill 包
1. 在弹出的"上传技能"窗口中
2. 将 `周报_SKILL.zip` 文件拖拽到上传区域
3. 或者点击上传区域选择文件

### 步骤 3: 确认上传
1. 系统会自动检测 `SKILL.md` 文件
2. 确认技能信息：
   - **名称**: 周报月报自动生成器
   - **描述**: 从 Git 提交记录、任务系统、文档变更中自动提取工作成果，一键生成结构化周报/月报
   - **版本**: 1.0.0
   - **分类**: 效率提升
3. 点击 **"确认"** 按钮

### 步骤 4: 安装完成
1. 等待上传和处理完成
2. 技能会出现在"已安装"列表中
3. 现在可以在 SOLO 中使用这个 Skill 了！

## 使用方式

### 方式一：直接对话（推荐）

在 SOLO 中直接输入：

```
帮我生成本周的周报
```

```
生成 5 月份的月报
```

```
我想看上周的工作总结，从 5 月 6 日到 5 月 12 日
```

### 方式二：使用命令行工具

如果你下载了 Skill 到本地，可以在项目目录中使用：

```bash
# 生成周报
python cli.py --author "你的名字" --team "你的团队" --output weekly_report.md

# 生成月报
python cli.py --type monthly --output monthly_report.md

# 指定日期范围
python cli.py --start-date 2026-05-01 --end-date 2026-05-31

# 使用配置文件
python cli.py --config config.json --output report.md
```

### 方式三：Python 代码调用

```python
from weekly_report_generator import ReportGenerator, ReportConfig

# 创建配置
config = ReportConfig(
    report_type="weekly",
    author_name="张三",
    team_name="开发团队",
    highlights=["完成核心功能开发"],
    challenges=["需要优化数据库"],
    plans=["推进新功能"]
)

# 生成报告
generator = ReportGenerator(config)
report = generator.generate()

# 保存报告
with open("weekly_report.md", "w", encoding="utf-8") as f:
    f.write(report)
```

## 常见问题

### Q1: 上传时提示"未包含 SKILL.md 文件"

**解决方案**:
- 确保上传的是 zip 文件
- 确保 zip 文件的根目录包含 `SKILL.md` 文件
- 使用我们提供的 `周报_SKILL.zip` 文件

### Q2: 安装后无法使用

**解决方案**:
1. 检查 Python 环境是否正确安装
2. 检查 Git 是否已安装并配置
3. 重启 SOLO 应用

### Q3: 生成的报告是空的

**可能原因**:
- 当前目录不是 Git 仓库
- 指定日期范围内没有提交记录

**解决方案**:
- 确保在项目根目录（包含 `.git` 文件夹）运行
- 使用 `--start-date` 和 `--end-date` 指定正确的日期范围

### Q4: 如何更新 Skill？

**步骤**:
1. 在 SOLO 技能管理中卸载旧版本
2. 重新上传新版本的 zip 文件
3. 或者在"已安装"列表中点击更新（如果有更新按钮）
